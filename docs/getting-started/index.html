    Getting Started¶

    This guide explains how to download and run the TicketSystem on your own machine. You need Docker Desktop or Podman Desktop — nothing else.

    Prerequisites¶

    Install one of the following:

    Platform Option A (recommended) Option B
    Windows Docker Desktop Podman Desktop
    macOS Docker Desktop Podman Desktop
    Linux sudo apt install docker.io docker-compose-v2 sudo apt install podman podman-compose

    Windows users

    Docker Desktop is the easiest option on Windows. It includes everything you need. Make sure WSL 2 is enabled (Docker Desktop will prompt you if it isn't).

    After installation, verify it works by opening a terminal and running:

    docker --version        # or: podman --version
docker compose version  # or: podman compose version

    1. Download the deploy folder¶

    Download the three deployment files from the repository and place them in a folder on your machine:

    • docker-compose.yml
    • deploy.sh (Linux / macOS)
    • deploy.ps1 (Windows)

    Or clone the whole repository and navigate to the deploy/ directory:

    git clone https://github.com/saskia-peters/schirrmeisterei.git
cd schirrmeisterei/deploy

    2. Start the application¶

    Windows (PowerShell)¶

    cd path\to\deploy
.\deploy.ps1

    Execution policy

    If PowerShell blocks the script, run this once first: 

    Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

    Linux / macOS (Bash)¶

    cd path/to/deploy
chmod +x deploy.sh
./deploy.sh

    What happens on first run¶

    1. A .env file is generated with random database passwords and a secret key — no manual configuration needed.
    2. Three container images are pulled from the internet:
      • PostgreSQL 18 — the database
      • TicketSystem Backend — the API server (runs database migrations automatically)
      • TicketSystem Frontend — the web interface
    3. All three containers start up. The backend waits for the database to be healthy, then applies migrations (creates tables, seeds initial data).

    This takes about 1–2 minutes the first time (image download). Subsequent starts take a few seconds.

    3. Open the application¶

    Once started, open your browser:

    Service URL
    Web application http://localhost:8080
    API documentation http://localhost:8080/api/docs

    Default login¶

    Field Value
    Email admin@example.com
    Password admin

    Change the password

    You will be prompted to change the password on first login. The default credentials are for initial setup only.

    4. Manage the application¶

    All commands work on every platform. Use deploy.ps1 on Windows or deploy.sh on Linux/macOS.

    Command Description
    .\deploy.ps1 / ./deploy.sh Start the application
    .\deploy.ps1 stop / ./deploy.sh stop Stop all containers
    .\deploy.ps1 update / ./deploy.sh update Pull latest version and restart
    .\deploy.ps1 logs / ./deploy.sh logs Follow live logs
    .\deploy.ps1 status / ./deploy.sh status Show container status

    5. Configuration (optional)¶

    The generated .env file in the deploy/ directory controls all settings. Edit it with any text editor:

    # Ports exposed on your machine
FRONTEND_PORT=8080
BACKEND_PORT=8000

# Database credentials (auto-generated, change if needed)
POSTGRES_USER=ticketsystem
POSTGRES_PASSWORD=<auto-generated>
POSTGRES_DB=ticketsystem

# Application secret for JWT tokens (auto-generated)
SECRET_KEY=<auto-generated>

# Max file upload size in MB
MAX_UPLOAD_SIZE_MB=10

# Pin a specific version instead of latest
# TICKETSYSTEM_VERSION=1.0.0

    After editing .env, restart the application:

    ./deploy.sh stop
./deploy.sh

    Data persistence¶

    All data is stored in Docker/Podman volumes and survives container restarts:

    Volume Contents
    postgres-data Database (users, tickets, settings, organisations)
    backend-uploads Uploaded file attachments and avatars

    Removing volumes deletes all data

    Running docker compose down -v (with -v) permanently deletes the database and all uploaded files. Only use this if you want a fresh start.

    Troubleshooting¶

    Containers don't start¶

    Check the logs for error messages:

    ./deploy.sh logs     # or: .\deploy.ps1 logs

    Port already in use¶

    Edit .env and change FRONTEND_PORT or BACKEND_PORT to an unused port:

    FRONTEND_PORT=9090
BACKEND_PORT=9000

    Reset everything (fresh start)¶

    Stop the application and remove all data:

    docker compose down -v   # or: podman compose down -v
rm .env
./deploy.sh              # re-generates secrets, creates fresh database

    Windows: "compose" not found¶

    Make sure Docker Desktop is running. The docker compose command is included with Docker Desktop but only works when the Docker engine is active (check the system tray icon).

    Podman on Windows¶

    If using Podman Desktop on Windows, make sure the Podman machine is running:

    podman machine start
